# 보안 개선 메모

이 프로젝트는 로컬 학습 및 포트폴리오 목적의 PHP + MySQL 쇼핑몰 예제입니다. 실서비스 수준의 모든 보안 통제가 완성된 상용 시스템으로 표현하지 않습니다.

## 적용된 주요 개선

| 구분 | 내용 |
| --- | --- |
| 관리자 인증 | 관리자 접근 확인은 `login_main_check.php`의 PHP session 기반 인증을 사용합니다. |
| 설정 분리 | DB/관리자 계정 정보는 `config.php.example` 기반 설정 분리와 환경변수 fallback 구조를 사용합니다. |
| Prepared Statement | 주문 저장, 상품 등록/수정/삭제, 관리자 회원/옵션/FAQ/주문 변경 처리의 주요 구간에 Prepared Statement 또는 정수 검증을 적용했습니다. |
| 출력 escaping | 관리자 상품 목록의 검색어와 DB 출력값에 HTML escaping을 보강했습니다. |
| 상품 이미지 처리 | 상품 수정 시 기존 이미지명은 조작 가능한 hidden input이 아니라 DB에서 조회한 현재 이미지명을 기준으로 삭제/교체합니다. |
| 관리자 POST+CSRF | 관리자 상품/회원/옵션/소옵션/FAQ/주문 변경 처리 파일은 주요 관리자 변경 요청 중심으로 POST 요청과 session 기반 CSRF 토큰 검증을 적용했습니다. |
| 오류 노출 축소 | SQL 전체를 사용자 화면에 출력하는 패턴을 주요 구간에서 제거하고 내부 오류는 `error_log()` 대상으로 정리했습니다. |

## 관리자 POST+CSRF 적용 범위

관리자 영역은 상품/회원/옵션/소옵션/FAQ/주문처럼 데이터가 변경되는 주요 처리 파일을 우선 대상으로 정리했습니다. 해당 처리 파일은 CSRF 검증 이후 입력값을 `$_POST` 기준으로 읽도록 맞췄습니다.


- 상품: 등록, 수정, 삭제
- 회원: 수정, 삭제
- 옵션: 등록, 수정, 삭제
- 소옵션: 등록, 수정, 삭제
- FAQ: 등록, 수정, 삭제
- 주문: 상태 변경, 삭제

## 실습 보조 영역 정리 범위

`juso/`, `sj/`는 포트폴리오의 핵심 쇼핑몰 흐름이 아니라 PHP/MySQL CRUD 실습 보조 영역입니다. 이번 정리에서는 short_open_tag 의존성, 숫자 ID 캐스팅, SQL 오류 화면 노출, 일부 문자열 입력 escaping과 redirect 방식을 최소 범위로 보완했지만, 핵심 쇼핑몰 기능과 같은 수준으로 완전 정비한 것은 아닙니다.

사용자 영역 전체 CSRF 토큰 적용은 아직 완료되지 않았으며, 관리자 주요 변경 처리 파일 중심으로 우선 적용된 상태입니다.

## 아직 남은 한계

- 회원 비밀번호 해시화가 전체 가입/수정/로그인 흐름에 완전 적용되어 있지 않을 수 있습니다.
- 전체 SQL 구간의 Prepared Statement 전환은 진행 중이며, 오래된 실습 보조 영역에는 문자열 SQL이 일부 남아 있을 수 있습니다.
- 실결제 PG 승인 검증, 위변조 방지, 결제 취소/환불 검증은 구현되어 있지 않습니다.
- 개인정보 암호화와 접근 로그/감사 로그 체계는 운영 배포 수준으로 구성되어 있지 않습니다.
- 운영 배포용 보안 헤더, HTTPS 강제, 쿠키 보안 속성 정책은 별도 점검이 필요합니다.
